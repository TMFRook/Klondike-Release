Wait, What is this Thing?

Do you know what the .NET Framework is? Do you know what NuGet is? No? If not, you can probably stop reading now. Instead perhaps do some research on monkeys riding dogs.

Klondike is similar to the NuGet.Server project that lets you host a package feed on your own private network. Klondike is different because it's faster. Way faster. The problem with NuGet.Server is that it slows down to a crawl after you add around 100 packages. Klondike uses Lucene.Net.Linq to index your projects to give you all the searchy goodness of Lucene along with super fast package retrieval.

You Should Use Klondike if...

  • You develop software with Microsoft.NET or the Mono Framework
  • You use NuGet
  • You don't want your build to break when nuget.org goes down, or
  • You create packages for private consumption, or
  • You want a place to push packages to test them before releasing to nuget.org

You should not use Klondike if you are not awesome.
